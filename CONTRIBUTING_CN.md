# 贡献

## 问题报告

- 检查该问题是否已被报告过。
- 检查该问题是否已在最新版本中修复。
- 清晰简明地描述您的问题。
- 友好和尊重对待他人。
- 包含任何相关的代码片段或错误信息。
- 使用语法正确、完整的句子打开新问题。

## 拉取请求

- 同一分支上没有冲突。
- PR 必须良好且写得清晰。
- 提交必须是原子性的，并具有清晰、描述性的英文消息。
- 允许在开发分支上提交。

## 代码风格

- 使用清晰、描述性的变量名。
- 编写清晰、简洁的注释。
- 使用 DDOX 风格注释来文档化 D 代码。
- 使用 doxygen 风格注释来文档化 C/C++ 代码。
- 宏以 `OE_` 开头。
- 每个 C/C++ 接口必须写在 `oe::<module>::impl` 中。
- `oe::<module>::details` 仅供内部使用。
- 每个 C/C++ 函数必须在其声明中添加 `OE_API` 宏。
- 使用 UTF-8 编码。
- 变量和函数命名采用蛇形命名法（snake case）。
- 类的成员变量应以 `m_` 前缀命名。
- 缩进应为 4 个空格。
- 使用 clang-format（C/C++）和 dfmt（D）来格式化代码。
- 在使用像 std::uint8_t 这样的类型时，使用 `types` 库。
- 使用 `scripts/add_module.sh` 来添加新模块。
- 在添加新模块配置时，在 [metas/configs.json](metas/configs.json) 中添加配置。
